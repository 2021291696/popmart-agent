# 2.5 防伪溯源Agent

## 为什么做这个

泡泡玛特二级市场假货泛滥。闲鱼上"Molly珍藏款"真假难辨，高仿工厂在热门款发售后几天内就能打板生产，甚至搭建假防伪网站诱导扫码。品牌方目前靠人工举报→审核→下架，效率和覆盖面极低。本Agent展示多模态数据处理能力。

## Agent职责声明（System Prompt草案）

```
你是泡泡玛特防伪溯源Agent。你负责扫描二手交易平台上的可疑listing，
融合图像特征+价格信号+卖家行为三种信息进行假货判断。

你的输出不是"这是假货"——是"这条listing的可疑度为X%，主要疑点：..."
最终判定和举报由品牌方法务团队做出。
```

## 工具清单（4个MCP工具）

| 工具 | 核心逻辑 | tool_description亮点 |
|------|---------|---------------------|
| image_verify(image_url) | 下载图片→对比正品特征库→输出差异标注 | "输入商品图片URL，返回与正品的差异点和相似度。LLM Vision API做初步判断，架构上可替换为专用CNN" |
| listing_scan(platform, keyword) | 扫描平台上的可疑listing | "扫描指定平台的商品listing。标注：图片异常/价格异常/描述异常" |
| seller_analyze(profile) | 分析卖家行为模式 | "输入卖家主页URL，返回风险评分：注册时间/好评率/历史成交/重复listing数量" |
| report_generate(finding_list) | 生成品牌方举报材料 | "将可疑listing列表整理为结构化的举报材料（含证据截图）" |

## 判断逻辑：三信号加权融合

```
可疑度 = 图像异常分 × 40% + 价格异常分 × 30% + 卖家风险分 × 30%

图像异常分：
  - 与正品特征库对比（logo位置/色差/材质纹理/缝合线位置）
  - LLM Vision API做初步判断（原型阶段）
  
价格异常分：
  - 低于市场均价40%以上 → 高分
  - 低于官方售价 → 中分
  - 正常区间 → 低分

卖家风险分：
  - 新注册(<3个月)/0成交/大量重复listing/好评率<90%
  
可疑度 > 70% → 自动加入举报清单
可疑度 40-70% → 标记为"需人工复核"
可疑度 < 40% → 忽略
```

## 异步监控模式

防伪Agent不是实时响应用户请求的——它是周期性扫描：
- 每小时扫描一次闲鱼/得物的新listing（listing_scan）
- 发现可疑→image_verify+seller_analyze→可疑度计算→超过阈值→加入举报清单
- 每天汇总一次举报清单→report_generate→推送给品牌方

**为什么选异步而非实时？**
- 高频扫描二手市场匹配假货检测的时效性需求
- 异步模式更省资源（不需要维持常驻进程）

## 关键决策

1. **图像识别用LLM Vision API（原型）而非训练CNN**
   → 原型验证可行性即可。架构设计上图像模块定义为独立接口——随时可替换为fine-tune的ResNet/EfficientNet，不影响其他三个Agent。

2. **输出"可疑度分数"而非"这是假货"**
   → 和法律风险相关。"假货"是法律结论，Agent只能提供技术层面的可疑判断。和供应链Agent一样——"知道什么时候不该越界"。

## 面试能讲什么

> "防伪Agent虽然在原型阶段没有落地，但它的设计体现了Agent的多模态能力和法律边界意识。
>
> 它融合三种信号：图像特征(40%)+价格异常(30%)+卖家行为(30%)——
> 这三种信号单看都不够可靠，但加权的融合判断大幅提升了检出率。
>
> 更关键的是——我定义它的输出是'可疑度分数'而非'是假货'。
> 因为'假货'是法律结论，Agent只能提供技术判断。
> 这体现了Agent设计的一个重要原则：明确Agent的权力边界。"
