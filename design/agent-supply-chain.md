# 2.3 供应链Agent

## 为什么做这个

泡泡玛特的盲盒配货今天靠人工经验判断——但消费者偏好在实时变化，人工跟不上。供应链Agent利用二手市场价格+社交热度+竞品动态等外部信号，给出配货调整建议。不直接操作ERP——它是辅助决策，不是自动执行。

## Agent职责声明（System Prompt草案）

```
你是泡泡玛特供应链分析Agent。你的职责是基于外部市场信号给出配货建议，
不做最终决策（最终决策由人工做出）。

你的核心逻辑：二手溢价率=需求强度，社交热度=需求趋势，竞品动态=分流判断。

## 工具
- demand_signal_scan: 扫描指定SKU/品类的市场信号
- inventory_optimize: 基于信号输出配货建议
- price_monitor: 监控二手平台价格波动
- alert_anomaly: 价格/库存异常告警
```

## 工具清单（4个MCP工具）

| 工具 | 核心逻辑 | tool_description亮点 |
|------|---------|---------------------|
| demand_signal_scan(region, ip_name) | 扫描该区域该IP的二手溢价率+求购帖数量+缺货报告 | "输入区域和IP名，返回需求信号四维度数据" |
| inventory_optimize(sku_list, signals) | 基于信号输出配货建议：热门款→增配，冷门款→减配或捆绑 | "输出'建议'而非'指令'——每个建议附带数据支撑和置信度" |
| price_monitor(platform, keyword) | 监控闲鱼/得物/千岛上的价格波动 | "追踪价格趋势+标注异常波动+对比官方售价" |
| alert_anomaly(metric, threshold) | 当指标超阈值时发出告警 | "库存天数>180天或溢价率<-30%时触发" |

## 决策逻辑

```
热门款信号 = 二手溢价率↑(>20%) + 求购帖数量↑ + 官方缺货率↑
  → 建议：该SKU增加配货比例X%（附信号数据）

冷门款信号 = 二手折价率↑(>30%) + 库存周转天数↑(>120天) + 社交热度↓
  → 建议：该SKU减少配货Y%或启动捆绑促销
```

## 关键决策

1. **选"建议型"而非"自动执行型"** — 供应链决策涉及实际库存变动，Agent做自动执行风险太高。"知道什么时候不该行动"和"知道什么时候该行动"同样重要。

2. **不直接操作ERP** — 原型阶段只做"外部视角"的配货建议。面试时能讲清楚生产环境的对接方案（API对接ERP系统+人工审批阈值）。

## 面试能讲什么

> "供应链Agent在四个Agent里最体现Agent的'决策边界'设计——它只给建议，不做自动执行。因为配货变动涉及实际库存，错误决策的代价太高。这是Agent设计的核心原则：不是把所有决策都交给Agent，而是明确每个决策的'人类在环'位置。"
