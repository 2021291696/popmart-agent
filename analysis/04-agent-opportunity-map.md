## 最终落地选择

**节点3只落地2个Agent原型：**

### 1. IP情报Agent（P0）🏆

**为什么选它**：
- 直击泡泡玛特最大的风险——LABUBU一个IP占38.1%营收
- 展示Agent的核心能力：多源信息整合 → 情感分析 → 自动告警
- 技术难度适中（2周可完成原型）：社交媒体搜索 + LLM情感分析 + 趋势判断
- 面试能讲"ReAct循环"的完整案例

**核心能力展示**：工具调用 + 多步推理 + 持续监控 + 自动决策

### 2. 消费者洞察Agent（P0）🏆

**为什么选它**：
- 展示Agent的另一个核心能力：RAG知识库 + 智能路由
- 技术难度适中：RAG流水线（Embedding→ChromaDB→检索→生成）
- 面试能讲"RAG流水线+评估体系"的完整案例
- 评估数据（metrics.json）是面试最有杀伤力的差异化展示

**核心能力展示**：RAG + 向量检索 + 路由策略 + 量化评估

### 为什么不是供应链Agent和防伪Agent？

- **供应链Agent**需要接入泡泡玛特内部ERP/库存系统 → 原型阶段无法获取真实数据，只能做"外部信号+模拟建议"，面试讲起来不够硬
- **防伪Agent**的图像识别需要训练数据 → 原型阶段用LLM Vision API凑合，但准确率无法保证，面试时可能被追问"你这个精度能用于生产吗"
- 但两个Agent的**设计文档保留** → 面试时提到"我设计了4个Agent，落地了2个核心的，另外2个留了完整设计。如果给我更多时间，这是扩展方向" → 展现系统思维

---

## 关键决策

1. **P0只选2个不贪多** → 做深做透 > 做多做浅。一个有评估数据的RAG系统比4个只有设计文档的Agent值钱得多。

2. **IP情报+消费者洞察** → 分别对应Agent的两大核心能力：
   - IP情报Agent = **外部信息获取**（搜索+抓取+分析+告警）
   - 消费者洞察Agent = **内部知识检索**（RAG+路由+引用来源+评估）
   - 面试时两个Agent一起讲 = 完整覆盖Agent的能力光谱

3. **选型标准不只是"技术可行性"** → 更看重"面试能讲出什么"。IP情报Agent能讲ReAct循环的故事，消费者洞察Agent能讲RAG评估的数据——这两样是面试官最想听到的。
